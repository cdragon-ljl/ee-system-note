## 一、内容来源

- 用户提供 Bilibili / b23.tv 视频链接
- 用 bilibili-research-kit 技能提取字幕
- 字幕优先用 AI 生成字幕（如有），次选用户上传字幕
- **所有笔记内容必须有字幕原文支撑，不得凭空编造**

---

## 二、内容准确性要求

| 规则 | 说明 |
|------|------|
| ✅ **正确性** | 所有技术知识点必须来自字幕原文，寄存器值、函数名、宏定义值等需核实 |
| ⚠️ **区分教学简化 vs 事实错误** | 讲师可能为教学做简化（如近似 key code 值），笔记中可注明"讲师举例"或直接使用标准值 |
| ❌ **禁止脑补** | 字幕未提及的内容不得自行添加 |
| 🔍 **存疑标注** | 若有不确定的地方，标注 `[待核实]` 并说明原因 |
| 📚 **引用原文** | 关键概念建议引用讲师原话作为支撑 |
| 🚫 **宏定义查表** | key code 等标准宏定义值，以 Linux 内核头文件 `linux/input-event-codes.h` 为准，讲师举例仅作参考 |

### 举例如下

| 情况 | 处理方式 |
|------|---------|
| 讲师说"KEY_POWER 值是29"（实际是116） | ✅ 使用标准值并注明"内核实际定义为 KEY_POWER=116" |
| 讲师说"type=1是按键事件"（正确值） | ✅ 直接使用 |
| 字幕未提及 select/poll 用法 | ❌ 不得自行补充，可在"拓展学习"部分标注"视频未涉及" |

## 四、公众号发布格式

### 公众号版
- Markdown 文件 + 使用 render.py 转为 HTML（wechat-formatter 编辑器）
- 主题：serene-blue（或其他选中的主题）
- 可包含：代码块、表格、流程图说明
- 作者署名：学嵌入式的长路

### 小红书版（可选）
- 用户确认后再决定是否出小红书版

## 五、工作流程

1. 用户发送 B站 视频链接
2. 提取字幕 → 保存到 `video2notes/` 目录（transcript 文件）
3. 生成学习笔记 → 保存到 `video2notes/` 目录
   - 命名规则：`video2notes/{系列}-{序号}-{英文标题}-{YYYYMMDD}.md`
   - 如：`video2notes/imx6ull-drv-01-arm-chip-to-operating-system-20260713.md`
4. 发送文件给用户审查
5. 用户确认后，渲染为公众号 HTML
